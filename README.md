# 局域网多用户聊天系统

## 系统概述

本系统是一个基于TCP协议的局域网聊天软件，支持**4台电脑**在同一局域网下协同工作：
- **1台服务器**：负责消息路由和用户管理（无权访问用户聊天记录）
- **3台客户端**：用户终端，支持群聊和私聊功能

系统采用C/S架构，使用ncurses库提供终端分屏界面，确保良好的用户体验。

## 主要特性

✅ **隐私保护**：服务器仅负责消息转发，无法查看或存储聊天内容  
✅ **群聊功能**：所有在线用户可参与群聊  
✅ **私聊功能**：任意两用户间可进行私密对话  
✅ **实时更新**：在线用户列表实时刷新  
✅ **友好界面**：三窗格终端界面，清晰易用  
✅ **跨平台**：支持Linux系统（Ubuntu 24.04测试通过）

## 系统架构

```
+----------------+     +----------------+     +----------------+     +----------------+
|   客户端A      |     |   客户端B      |     |   客户端C      |     |   服务器       |
| (用户1)        |     | (用户2)        |     | (用户3)        |     | (消息路由)     |
|                |     |                |     |                |     |                |
| 在线用户列表   |     | 在线用户列表   |     | 在线用户列表   |     | 用户管理       |
| 聊天记录       |     | 聊天记录       |     | 聊天记录       |     | 连接处理       |
| 消息输入       |     | 消息输入       |     | 消息输入       |     |                |
+----------------+     +----------------+     +----------------+     +----------------+
        |                      |                      |                      |
        |                      |                      |                      |
        +----------------------+----------------------+----------------------+
                                   TCP/IP 网络通信 (端口: 8888)
```

## 部署要求

### 硬件要求
- 4台电脑（1台服务器 + 3台客户端）
- 所有电脑必须在同一局域网内
- 每台电脑至少512MB内存

### 软件要求
- **操作系统**：Linux（推荐 Ubuntu 20.04/24.04）
- **编译器**：GCC 7.0 或更高版本
- **依赖库**：
  - `libncurses-dev`（终端界面库）
  - `pthread`（多线程支持）

### 安装依赖（Ubuntu/Debian）
```bash
sudo apt update
sudo apt install build-essential libncurses-dev
```

## 编译与安装

### 1. 克隆或下载项目
```bash
git clone <your-repo-url>  # 如果从GitHub克隆
# 或直接使用当前目录
cd /path/to/chat-system
```

```bash

```

编译完成后将生成两个可执行文件：

- `client`：客户端程序

### 3. 清理编译文件（可选）
```bash
make clean
```

## 使用方法

### 步骤1：启动服务器（在服务器电脑上执行）

1. 打开终端，进入项目目录
2. 启动服务器：
   ```bash
   ./server
   ```
3. 记录显示的服务器IP地址（例如：`192.168.1.100`）

**服务器输出示例：**
```
=== 局域网聊天服务器启动成功 ===
服务器IP: 192.168.1.100
监听端口: 8888
等待客户端连接...
================================
```

### 步骤2：启动客户端（在3台客户端电脑上分别执行）

1. 在每台客户端电脑上打开终端，进入项目目录
2. 启动客户端：
   ```bash
   ./client
   ```
3. 按提示输入服务器IP地址（步骤1中记录的IP）
4. 输入用户名（建议使用不同用户名以便区分）

**客户端启动示例：**
```
=== 局域网聊天客户端 ===
请输入服务器IP地址 (默认127.0.0.1): 192.168.1.100
请输入您的用户名: Alice
连接服务器成功！启动聊天界面...
```

### 步骤3：使用聊天功能

#### 群聊功能
- 直接在输入框中输入消息并按回车
- 消息将发送给所有在线用户
- 消息格式：`[用户名]: 消息内容`

#### 私聊功能
1. 在输入框中输入 `sendTo` 并按回车
2. 输入目标用户名（从左侧在线用户列表中选择）
3. 输入私聊消息
4. 私聊消息格式：`[PRIVATE] [用户名] -> 你: 消息内容`

#### 退出程序
- 在输入框中输入 `end` 并按回车
- 程序将自动清理资源并退出

## 故障排除

### 常见问题及解决方案

**1. 连接服务器失败**
- 检查服务器是否已启动
- 确认服务器IP地址正确
- 检查防火墙设置（确保端口8888开放）
- 确保所有电脑在同一局域网

**2. 无法显示中文**
- 确保终端支持UTF-8编码
- 设置环境变量：`export LANG=zh_CN.UTF-8`

**3. 界面显示异常**
- 确保终端窗口足够大（建议至少80x24字符）
- 某些终端模拟器可能不完全支持ncurses

**4. 消息延迟**
- 检查网络连接质量
- 确保服务器电脑性能足够

### 网络配置检查

```bash
# 检查服务器IP
ip addr show

# 测试端口连通性（从客户端）
telnet 192.168.1.100 8888

# 检查防火墙状态
sudo ufw status
```

## 安全说明

- **隐私保护**：服务器仅存储用户基本信息（用户名、IP），不存储聊天内容
- **明文传输**：消息以明文形式传输，建议在可信局域网内使用
- **无认证机制**：当前版本无需密码，适合内部使用
- **端口安全**：默认使用8888端口，请确保网络安全

# Web聊天服务器

这是一个基于Flask和SocketIO的现代化Web聊天服务器，支持群聊、私聊和多人群聊功能。

## 功能特性

- 群聊：所有在线用户都能看到的消息
- 私聊：用户之间的私人消息
- 多人群聊：创建特定群组进行交流

## 环境要求

- Python 3.7+
- Linux/WSL 环境

## 安装与运行

### 方法一：使用启动脚本（推荐）

```bash
./run_server.sh
```

### 方法二：手动启动

1. 激活虚拟环境：
```bash
source ./chat_env/bin/activate
```

2. 运行服务器：
```bash
python web_server.py
```

## 访问应用

启动后，在浏览器中打开 `http://localhost:5000` 即可开始使用聊天室。

## 项目结构

- `web_server.py` - 主服务器程序
- `templates/index.html` - 前端界面
- `chat_env/` - Python虚拟环境
- `run_server.sh` - 启动脚本
